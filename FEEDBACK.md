# FEEDBACK — 10 小白验证 / 10-Novice Validation

> 这是 demo-factory 的生死测试。这条规则发布时会写进 README:**如果 10 个真实小白中,少于 3 人能在无人帮助的情况下走到一个能点的 demo,这个项目就停止。**
> This is the live-or-die test. The rule (to be carried in the README at release): **if fewer than 3 of 10 real novices reach a working demo unaided, this project stops.**

## 找什么样的人 / Who to recruit

- 10 个**从没写过程序、也从没用过 AI 编程工具**的成年人(店主、家长、退休老师、亲戚都行)。
- 至少 6 人以中文为主要语言(产品默认 zh)。
- 不要找"会装电脑软件的朋友"帮忙型选手 —— 要平时只用微信和浏览器的人。

## 测试怎么做 / How to run one session

1. 在项目目录运行 `npm start`,然后帮他们打开浏览器到 `http://localhost:3210`(这一步可以代劳 —— 我们测的是向导,不是装机)。
2. 只说这一句,然后闭嘴:
   > "这个网页说它能把你的一个想法变成一个能用的小东西。你随便试试。"
3. **不许帮忙。** 他们问问题就回答:"你觉得该点哪就点哪。" 全程计时,默默记录下面的观察表。
4. 结束后问 3 个问题(下面的"事后三问")。

## 事后三问 / The three questions afterward

1. "你刚才拿到的三个做法,用你自己的话说说它们有什么不一样?"(测选项是否真的被理解)
2. "这个做出来的东西,你会给谁看?"(测 demo 是否值得炫耀 —— 说不出人名 = 没过)
3. "如果要花 10 块钱再做一个,你会做什么?"(测付费意愿方向,不承诺任何功能)

## 什么算信号 / What counts as signal

| 信号 | 阈值 |
|---|---|
| **走到 demo(无助)** | **≥3/10,否则项目停止(kill criterion)** |
| 能说清 3 个选项的区别 | ≥5/10 算"选项页有效" |
| 会把 demo 给别人看(说得出具体的人) | ≥3/10 算"wow 成立" |
| 主动问"还能做别的吗" | 每一个都记下来 —— 这是市场信号 |
